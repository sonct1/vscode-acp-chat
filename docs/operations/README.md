# Operations Documentation

Use this area for running the service: configuration, environments, deployment, rollback, monitoring, alerting, troubleshooting, recovery, and incident runbooks.

Typical files:

- `configuration.md` — runtime configuration and environment variables.
- `deployment.md` — deploy process and environment notes.
- `rollback.md` — rollback/recovery steps.
- `troubleshooting.md` — common checks and fixes.
- `runbooks/<symptom-or-alert>.md` — actionable runbooks for alerts/incidents.

Do not put product requirements, API payload details, technical design alternatives, or implementation task status here.
