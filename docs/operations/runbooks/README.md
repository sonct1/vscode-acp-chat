# Operational Runbooks

Use this area for actionable incident/alert runbooks with symptoms, checks, commands, mitigation, rollback/recovery, and escalation.

Typical files:

- `<alert-name>.md`
- `<symptom-name>.md`
- `<dependency-name>-outage.md`
- `<integration-issue>.md` — production integration debugging/recovery for SDK, API, event, webhook, or downstream dependency issues.

Do not put general architecture, feature requirements, or implementation plans here.
