# Architecture Documentation

Use this area for service-wide architecture: module boundaries, runtime topology, storage, messaging, external dependencies, security model, major data flows, and cross-cutting constraints.

Typical files:

- `README.md` — architecture overview and routing.
- `context.md` — service context, actors, upstream/downstream systems.
- `runtime.md` — deployment/runtime topology.
- `data-flow.md` — major synchronous/asynchronous data flows.
- `acp-chat-layout.md` — ACP Chat webview wireframe and source-code component map.
- `adr/ADR-XXXX-<decision>.md` — durable architecture decisions.

Do not put feature-specific implementation checklists, endpoint payload details, release plans, or troubleshooting runbooks here.
