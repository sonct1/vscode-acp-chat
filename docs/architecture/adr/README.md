# Architecture Decision Records

Use this area for durable architecture decisions with context, alternatives, trade-offs, and consequences.

Typical files:

- `ADR-0001-<decision-title>.md`
- `ADR-0002-<decision-title>.md`

Use ADRs when a decision is hard to reverse, affects multiple modules/teams, changes operational behavior, or explains why a design chose one option over alternatives.

Do not put generic meeting notes, task lists, implementation plans, or decisions that are obvious from code here.
