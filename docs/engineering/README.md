# Engineering Documentation

Use this area for service engineering conventions: build, test, lint, code style, branching, database migrations, logging, tracing, dependency management, and local development standards.

Typical files:

- `testing.md` — test layers, naming, fixtures, and verification commands.
- `coding-conventions.md` — service-specific coding rules.
- `database-migration.md` — schema migration conventions.
- `logging-observability.md` — logging, metrics, tracing, and correlation rules.
- `local-development.md` — local dependencies and developer workflow when too large for root `README.md`.

Do not put product requirements, API contracts, feature backlog, implementation plans, or incident runbooks here.
