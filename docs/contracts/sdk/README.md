# SDK Documentation

Use this area for consumer-facing SDK documentation: installation, configuration, authentication, usage examples, error handling, compatibility, versioning, and migration guides for SDKs provided by this service.

Typical files:

- `installation.md` — package coordinates, supported versions, and setup.
- `configuration.md` — required configuration, endpoints, timeouts, and defaults.
- `authentication.md` — auth headers, tokens, credentials, and principal propagation.
- `usage.md` — common use cases and recommended call patterns.
- `examples.md` — runnable or copyable code examples.
- `error-handling.md` — exceptions, error mapping, retry, idempotency, and fallback guidance.
- `versioning.md` — compatibility policy, deprecation policy, and release notes.
- `migration-guide.md` — breaking changes and upgrade steps.

Do not put internal SDK implementation design, provider rollout plans, active task status, or production incident runbooks here.
