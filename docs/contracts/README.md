# Contract Documentation

Use this area for service interface contracts: REST API endpoints, request/response payloads, auth behavior, error codes, compatibility notes, OpenAPI specs, events, webhooks, and examples.

Typical files:

- `openapi.yaml` — OpenAPI contract when applicable.
- `error-codes.md` — shared service error code catalog.
- `events.md` — published/subscribed event contracts when applicable.
- `webhooks.md` — webhook contracts when applicable.
- `endpoints/<resource>.md` — endpoint-level contract and examples.
- `sdk/README.md` — consumer-facing SDK documentation when this service provides an SDK.

Do not put product requirements, internal implementation details, deployment steps, or task plans here.
