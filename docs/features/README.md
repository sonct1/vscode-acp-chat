# Feature Documentation

Use this area as a catalog for durable feature context when a feature spans product scope, technical design, API contract, rollout, or operational concerns.

Typical files:

- `README.md` — feature catalog and routing.
- `FEAT-NNNN-<slug>/README.md` — feature brief with links to related product, design, API, plan, and operations docs.
- `FEAT-NNNN-<slug>/rollout.md` — rollout notes when they remain useful after implementation.

Do not put backlog items, active task status, sprint planning, one-shot bug notes, or detailed endpoint payloads here. Use Beads for executable tasks and status.
