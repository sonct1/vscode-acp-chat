# Technical Design Documentation

Use this area for durable technical designs of complex service flows, modules, workers, integrations, data model changes, retry/idempotency behavior, security behavior, and error handling.

Typical files:

- `<flow>-design.md`
- `<module>-design.md`
- `<integration>-design.md`
- `<worker>-design.md`

Do not put product requirements, task checklists, implementation phase plans, endpoint payload details, or operational runbooks here.
